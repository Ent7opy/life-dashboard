# Life Dashboard – Documentation & Planning

This directory contains research, ideas, and tasks for the Life Dashboard project.

## Structure

- **research/**: Background research, findings, and analysis.
- **ideas/**: Product ideas, feature proposals, and improvement suggestions.
- **tasks/**: Development tasks for Claude (the developer). Each task is a separate markdown file with clear description and acceptance criteria.
- **decisions/**: Record of key product decisions and rationale.

## Purpose

As Product Owner, Sheldon maintains this documentation to guide development, capture context, and ensure continuity. Claude (the developer) will implement tasks from `tasks/`.

## How to Use

1. **Review research** before designing new features.
2. **Add ideas** as they emerge; discuss with stakeholders.
3. **Create tasks** for Claude when an idea is ready for implementation.
4. **Record decisions** to avoid ambiguity later.

## Current Status

- **Dashboard**: Live on GitHub Pages (`https://ent7opy.github.io/life-dashboard/`).
- **Backend API**: Under deployment on Railway (pending).
- **Primary features**: Interactive progress rings, weekly review, reading list, dark/light mode, Bulgarian/German/Hybrid path selector.
